# Learn Skill

Capture lessons from the current session and update CLAUDE.md.

## Instructions

1. Review the recent conversation for:
   - Mistakes that were corrected
   - Patterns that worked well
   - User preferences discovered
   - Conventions that should be enforced

2. Identify what should be documented:
   - New rules ("always do X", "never do Y")
   - Code patterns specific to this project
   - Clarifications to existing rules

3. Show the user what you learned and propose additions to CLAUDE.md

4. After user approval, update the appropriate section in CLAUDE.md:
   - Code style → "Code Style" section
   - SolidJS patterns → "SolidJS Patterns" section
   - Tooling/commands → "Commands" section
   - New category → Create new section if needed

5. Keep additions concise - one line per rule when possible

6. Do NOT add:
   - Generic best practices (only project-specific learnings)
   - Temporary fixes or workarounds
   - Anything already covered by existing rules
